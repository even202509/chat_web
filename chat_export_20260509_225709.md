账号停用涉及两处变更：

**1. 登录拦截**

`login()` 函数中，密码验证通过后多了一步 `is_active` 检查：

```python
if not user[0][4]:  # users 表第 5 列 = is_active (boolean)
    return {"success": False, "error": "Account is deactivated"}
```

`SELECT * FROM users` 返回的列顺序是 `id, username, password_hash, created_at, last_login, is_active`，所以 `user[0][4]` 就是 `is_active` 字段。如果值为 `FALSE`，直接拒绝登录。

**2. 自停用事件**

新增 `deactivate_my_account` Socket 事件：已登录用户调用后，`UPDATE users SET is_active = FALSE`，然后通过 Socket 通知自己的客户端 `account_deactivated`。

没有实现重新激活——因为停用后账号无法登录，重新激活需要管理员在数据库直接操作或另行实现。如果后续需要这个能力，可以再加一个管理后台接口。
